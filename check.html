Check Module

App settings, multiple items selection or just form input, there's several places to put checkboxes, radios and switches.
This module bring's the three of them at the same time, because they share the same base properties internally.

Notice: This components requires a label with the data-css-mark attribute after a native checkbox/radio to make it clickable using only CSS.

Checkbox

Radio

Switch

Example: HTML inline code <h5>Checkbox</h5> <div data-css-role="checkbox">  <input id="simple_check" type="checkbox">  <label data-css-mark for="simple_check">   Toggle me  </label> </div> <h5>radio</h5> <div role="radiogroup">  <div data-css-role="radio">   <input id="simple_radio1" name="simple_radios" type="radio">   <label data-css-mark for="simple_radio1">    Go in   </label>  </div>  <div data-css-role="radio">   <input id="simple_radio2" name="simple_radios" type="radio">   <label data-css-mark for="simple_radio2">    Go out   </label>  </div> </div> <h5>Switch</h5> <div data-css-role="switch">  <input id="simple_switch" type="checkbox">  <label data-css-mark for="simple_switch">   Enable it?  </label> </div>

Notice: We recommend wrapping multiple radios with the same name attribute in a role="radiogroup" container.

You can also use the ARIA attribute role instead our custom data-css-role, but you will have to manage the component's check state manually through the aria-checked attribute.
On the other hand, this way you may not need the native element, instead you can use a hidden input:

Example: HTML inline code <div aria-checked="mixed" role="checkbox">  <input id="aria_mixed" type="hidden" value="true">  <label data-css-mark>   I'm confused  </label> </div> <div aria-checked="false" role="radio">  <input type="hidden" value="false">  <label data-css-mark>   I'm off  </label> </div> <div aria-checked="true" role="switch">  <input type="hidden" value="true">  <label data-css-mark>   I'm ON  </label> </div>

Sizes

You can use four sizes, wich can be chosen by our data-css-size attribute.
This setting applies to the three components, but for simplicity we show only checkboxes bellow.

Example: HTML inline code <div data-css-size="small" data-css-role="checkbox">  <input id="small" type="checkbox">  <label data-css-mark for="small">   small  </label> </div> <div data-css-role="checkbox">  <input id="medium" type="checkbox">  <label data-css-mark for="medium">   medium (default)  </label> </div> <div data-css-size="big" data-css-role="checkbox">  <input id="big" type="checkbox">  <label data-css-mark for="big">   big  </label> </div> <div data-css-size="huge" data-css-role="checkbox">  <input id="huge" type="checkbox">  <label data-css-mark for="huge">   huge  </label> </div> <div data-css-size="full" data-css-role="checkbox">  <input id="full" type="checkbox">  <label data-css-mark for="full">   fullsize  </label> </div>

Forced states

There are two types of states you can use with this components:

  • Button states
    The of the button on background
  • Selection states
    The state of the active part

Button States

You may choose, if you wish, one of the for button states:

  • normal
  • selected
  • pressed
  • disabled

They are automatically chosen from the component current state in the page, but can also be forced by our data-css-state attributes.
Again, the setting applies to the three components, but for simplicity we show only switches bellow.

Example: HTML inline code  <div data-css-state="normal" data-css-role="switch">  <input type="hidden" value="false">  <label data-css-mark>  normal  </label> </div>  <div data-css-state="selected" data-css-role="switch">  <input type="hidden" value="false">  <label data-css-mark>  selected  </label> </div>  <div data-css-state="pressed" data-css-role="switch">  <input type="hidden" value="false">  <label data-css-mark>  pressed  </label> </div>  <div data-css-state="disabled" data-css-role="switch">  <input type="hidden" value="false">  <label data-css-mark>  disabled  </label> </div>

Selection States

You may use one of the three selection states:

  • unchecked
  • checked
  • mixed (checkbox only)

For that functionality we provide only ARIA attributes for now.

Example: HTML inline code <div aria-checked="false" role="radio">  <input type="hidden" value="false">  <label data-css-mark>   unchecked  </label> </div> <div aria-checked="true" role="switch">  <input type="hidden" value="true">  <label data-css-mark>   checked  </label> </div> <div aria-checked="mixed" role="checkbox">  <input type="hidden" value="mixed">  <label data-css-mark>   mixed (checkbox only)  </label> </div>

Accent Colors

You can use accent colors in all states and sizes.


Example: HTML inline code <div aria-checked="false" role="switch">  <input type="checkbox" id="accent_normal">  <label data-css-mark for="accent_normal">   unchecked  </label> </div> <div aria-checked="true" role="radio">  <input type="hidden" value="true">  <label data-css-mark>   checked  </label> </div> <div aria-checked="mixed" role="checkbox">  <input type="hidden" value="mixed">  <label data-css-mark>   mixed  </label> </div>
