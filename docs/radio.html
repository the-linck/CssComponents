Radio

App settings, single item selection or just form input, there's several places to put radios - like checkboxes.
To style this important element, there's Radio Component of course.

Notice: This component requires a label with the data-css-mark attribute after native radio to make the component clickable using only CSS.

Notice: Obviously, you must provide a common name attribute to the radios to group them.

Example: HTML inline code <div data-css-role="radio">  <input name="simple" id="plain_1" type="radio">  <label data-css-mark for="plain_1">   Choose me.  </label> </div> <div data-css-role="radio">  <input name="simple" id="plain_2" type="radio">  <label data-css-mark for="plain_2">   Choose you.  </label> </div>

You can also use the ARIA attribute role="radio" instead our custom data-css-role="radio", but you will have to manage the component's check state manually through the aria-checked attribute.
On the other hand, this way you may not need the native radios, instead you can use a hidden input:

Example: HTML inline code <div aria-checked="false" role="radio">  <input name="simple-aria-you" type="hidden">  <label data-css-mark>   It's not you  </label> </div> <div aria-checked="true" role="radio">  <input name="simple-aria-me" type="hidden">  <label data-css-mark>   It's me  </label> </div>

Sizes

You can use five radio sizes, wich can be chosen by our data-css-size attribute:

Example: HTML inline code <div data-css-size="small" data-css-role="radio">  <input name="size" type="radio">  <label data-css-mark>   small  </label> </div> <div data-css-role="radio">  <input name="size" type="radio">  <label data-css-mark>   medium (default)  </label> </div> <div data-css-size="big" data-css-role="radio">  <input name="size" type="radio">  <label data-css-mark>   big  </label> </div> <div data-css-size="huge" data-css-role="radio">  <input name="size" type="radio">  <label data-css-mark>   huge  </label> </div> <div data-css-size="full" data-css-role="radio">  <input name="size" type="radio">  <label data-css-mark>   fullsize  </label> </div>

States

There are two types of states you can use with this component:

  • Button states
    The of the button bellow the radio mark.
  • Selection states
    The state of the radiobutton mark

Button States

You may choose, if you wish, one of the for button states:

  • normal
  • selected
  • pressed
  • disabled

They are automatically chosen from the component current state in the page, but can also be forced by our data-css-state attributes:

Example: HTML inline code <div data-css-state="normal" role="radio">  <input name="state" id="state_normal" type="radio">  <label data-css-mark for="state_normal">   normal  </label> </div> <div data-css-state="aria-selected" role="radio">  <input name="state" id="state_selected" type="radio">  <label data-css-mark for="state_selected">   selected  </label> </div> <div data-css-state="pressed" role="radio">  <input name="state" id="state_pressed" type="radio">  <label data-css-mark for="state_pressed">   pressed  </label> </div> <div data-css-state="disabled" role="radio">  <input name="state" id="state_disabled" type="radio">  <label data-css-mark for="state_disabled">   disabled  </label> </div>

You can use also the following ARIA attributes, but for that you must provide the role and aria-checked attributtes to use 'em, like in the second example of this component.

  • aria-selected
  • aria-pressed
  • aria-disabled

Of course, using the HTML's standart disabled attribute will naturally show the radiobutton exactly like the custom data-css-state="disabled" state and aria-disabled="true".

Selection States

You may use one of the two radio selection states:

  • unchecked
  • checked

For that functionality we provide only ARIA attributes for now.


Example: HTML inline code <div aria-checked="false" role="radio">  <input type="hidden">  <label data-css-mark>   unchecked  </label> </div> <div aria-checked="true" role="radio">  <input type="hidden">  <label data-css-mark>   checked  </label> </div>

Accent Colors

You may use your chosen accent color to make a radio more visible to the user.

Example: HTML inline code <labeldata-css-size="small" role="radio" title="A simple radio">  <input name="size" type="radio">  <span data-css-mark></span>  small </label> <label role="radio" title="A simple radio">  <input name="size" type="radio">  <span data-css-mark></span>  medium (default) </label> <label data-css-size="big" role="radio" title="A simple radio">  <input name="size" type="radio">  <span data-css-mark></span>  big </label> <label data-css-size="huge" role="radio" title="A simple radio">  <input name="size" type="radio">  <span data-css-mark></span>  huge </label>

Differnt Colors

You can combine both primary and secondary colors in the same radiobutton.

Example: HTML inline code <div data-css-role="radio">  <input name="different" id="different_1" type="radio">  <label data-css-accent data-css-mark for="different_1">   Primary and accent  </label> </div> <div data-css-accent data-css-role="radio">  <input name="different" id="different_1" type="radio">  <label data-css-primary data-css-mark for="different_1">   Accent and primary  </label> </div>

Notice: As mentioned everywhere, we take primary color as the default, so there's no need to declare that the component uses the primary color for the button in the combination "Primary and accent".
