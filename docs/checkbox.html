Checkbox

App settings, multiple items selection or just form input, there's several places to put checkboxes.
To style this important element, there's Checkbox Component of course.

Notice: This component requires a label with the data-css-mark attribute after the native checkbox to make it clickable using only CSS.

Example: HTML inline code <div data-css-role="checkbox" title="A simple checkbox">  <input id="simple" type="checkbox">  <label data-css-mark for="simple">   Toggle me.  </label> </div>

You can also use the ARIA attribute role="checkbox" instead our custom data-css-role="checkbox", but you will have to manage the component's check state manually through the aria-checked attribute.
On the other hand, this way you may not need the native checkbox, instead you can use a hidden input:

Example: HTML inline code <div aria-checked="true" role="checkbox">  <input type="hidden" value="true">  <label data-css-mark>   I'm ON.  </label> </div> <div aria-checked="false" role="checkbox">  <input type="hidden" value="false">  <label data-css-mark>   I'm OFF.  </label> </div>

Sizes

You can use four checkbox sizes, wich can be chosen by our data-css-size attribute:

Example: HTML inline code <div data-css-size="small" data-css-role="checkbox">  <input id="small" type="checkbox">  <label data-css-mark for="small">   small  </label> </div> <div data-css-role="checkbox">  <input id="medium" type="checkbox">  <label data-css-mark for="medium">   medium (default)  </label> </div> <div data-css-size="big" data-css-role="checkbox">  <input id="big" type="checkbox">  <label data-css-mark for="big">   big  </label> </div> <div data-css-size="huge" data-css-role="checkbox">  <input id="huge" type="checkbox">  <label data-css-mark for="huge">   huge  </label> </div> <div data-css-size="full" data-css-role="checkbox">  <input id="full" type="checkbox">  <label data-css-mark for="full">   fullsize  </label> </div>

States

There are two types of states you can use with this component:

  • Button states
    The of the button bellow the check mark.
  • Selection states
    The state of the checkbox mark

Button States

You may choose, if you wish, one of the for button states:

  • normal
  • selected
  • pressed
  • disabled

They are automatically chosen from the component current state in the page, but can also be forced by our data-css-state attributes:

Example: HTML inline code  <div data-css-state="normal" data-css-role="checkbox">  <input type="hidden" value="false">  <label data-css-mark>  normal  </label> </div>

Selection States

You may use one of the three checkbox selection states:

  • unchecked
  • checked
  • mixed

For that functionality we provide only ARIA attributes for now.

Example: HTML inline code <div aria-checked="false" role="checkbox">  <input type="hidden" value="false">  <label data-css-mark>   unchecked  </label> </div> <div aria-checked="true" role="checkbox">  <input type="hidden" value="true">  <label data-css-mark>   checked  </label> </div> <div aria-checked="mixed" role="checkbox">  <input type="hidden" value="mixed">  <label data-css-mark>   mixed  </label> </div>

Accent Colors

You can use accent colors in all checkbox states and sizes.


Example: HTML inline code <div aria-checked="false" role="checkbox">  <input type="checkbox" id="accent_normal">  <label data-css-mark for="accent_normal">   unchecked  </label> </div> <div aria-checked="true" role="checkbox">  <input type="hidden" value="true">  <label data-css-mark>   checked  </label> </div> <div aria-checked="mixed" role="checkbox">  <input type="hidden" value="mixed">  <label data-css-mark>   mixed  </label> </div>

Differnt Colors

You can combine both primary and secondary colors in the same checkboxbutton.

Example: HTML inline code <div data-css-role="checkbox">  <input name="different" id="different_1" type="checkbox">  <label data-css-accent data-css-mark for="different_1">   Primary and accent  </label> </div> <div data-css-accent data-css-role="checkbox">  <input name="different" id="different_1" type="checkbox">  <label data-css-primary data-css-mark for="different_1">   Accent and primary  </label> </div>

Notice: As mentioned everywhere, we take primary color as the default, so there's no need to declare that the component uses the primary color for the button in the combination "Primary and accent".
