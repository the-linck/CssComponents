Resting label

When an custom input without a placeholder is empty, we move it's label above the field to indicate better what is meant to be inserted.
This functionality is not present in native input types because, currently, is impossible to know if a native field is empty with only CSS.

Example: HTML inline code <div data-css-group="textbox">  <div aria-labelledby="custom_label_resting" contenteditable="true" role="textbox">  </div>  <label id="custom_label_resting">Edit me out</label> </div>

Helper text

To give the user a context about the field, use an element with the data-css-type="info" attribute.
It's recommended to use short texts, one line if possible.

Type anything over here
Example: HTML inline code <div data-css-group="textbox">  <input id="native_text_helper" placeholder="Give me an e-mail" type="text">  <label for="native_text_helper">Some e-mail</label>  <div data-css-type="info">Test some e-mails here</div> </div>

Error message

It's important to inform the user when their input is invalid.
We provide that functionality by inserting an element with data-css-type="error" attribute after the input label, using the native HTML inpupt validation or the aria-invalid attribute.

We also provide support for standart and custom icons by the data-css-icon attribute.

Required and optional input

Indicating whether a field is optional or required is a good pratice.
To help you in doing in, we provide this functionality out-of-box. Just follow this convention:

  • required field
    Use required attibute for native input elements or aria-required for custom textbox.
  • optional field
    Use our data-css-optional attribute
Example: HTML inline code <div data-css-group="textbox">  <input id="native_text_require" placeholder="Heisenberg?" required type="text">  <label for="native_text_require">Say my name</label> </div> <div data-css-group="textbox">  <input data-css-optional id="native_text_optional" placeholder="Cook!" type="text">  <label for="native_text_optional">What do I do?</label> </div>

Notice: There's also out data-css-required attribute, avaliable to indicate that all textboxes in a container (a for, form example), or most of them, are required.

Icons

We also provide out-of-box support for icons alongside the textbox, be on the left or on the right side. All you have to do is put an element with data-css-icon and data-css-side *BEFORE* the input and we take care of it's size and positioning.
