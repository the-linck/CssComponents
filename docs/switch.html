Switch

App settings, multiple items selection or just form input, there's several places to put switches.
To style this important element, there's Switch Component of course.

Notice: This component requires a label with the data-css-mark attribute after the native switch to make it clickable using only CSS.

Example: HTML inline code <div data-css-role="switch" title="A simple switch">  <input id="simple" type="switch">  <label data-css-mark for="simple">   Toggle me.  </label> </div>

You can also use the ARIA attribute role="switch" instead our custom data-css-role="switch", but you will have to manage the component's check state manually through the aria-checked attribute.
On the other hand, this way you may not need the native switch, instead you can use a hidden input:

Example: HTML inline code <div aria-checked="true" role="switch">  <input type="hidden" value="true">  <label data-css-mark>   I'm ON.  </label> </div> <div aria-checked="false" role="switch">  <input type="hidden" value="false">  <label data-css-mark>   I'm OFF.  </label> </div>

Sizes

You can use four switch sizes, wich can be chosen by our data-css-size attribute:

Example: HTML inline code <div data-css-size="small" data-css-role="switch">  <input id="small" type="switch">  <label data-css-mark for="small">   small  </label> </div> <div data-css-role="switch">  <input id="medium" type="switch">  <label data-css-mark for="medium">   medium (default)  </label> </div> <div data-css-size="big" data-css-role="switch">  <input id="big" type="switch">  <label data-css-mark for="big">   big  </label> </div> <div data-css-size="huge" data-css-role="switch">  <input id="huge" type="switch">  <label data-css-mark for="huge">   huge  </label> </div> <div data-css-size="full" data-css-role="switch">  <input id="full" type="switch">  <label data-css-mark for="full">   fullsize  </label> </div>

States

There are two types of states you can use with this component:

  • Button states
    The of the button bellow the check mark.
  • Selection states
    The state of the switch mark

Button States

You may choose, if you wish, one of the for button states:

  • normal
  • selected
  • pressed
  • disabled

They are automatically chosen from the component current state in the page, but can also be forced by our data-css-state attributes:

Example: HTML inline code  <div data-css-state="normal" data-css-role="switch">  <input type="hidden" value="false">  <label data-css-mark>  normal  </label> </div>

Selection States

You may use one of the three switch selection states:

  • unchecked
  • checked

For that functionality we provide only ARIA attributes for now.

Example: HTML inline code <div aria-checked="false" role="switch">  <input type="hidden" value="false">  <label data-css-mark>   unchecked  </label> </div> <div aria-checked="true" role="switch">  <input type="hidden" value="true">  <label data-css-mark>   checked  </label> </div>

Accent Colors

You can use accent colors in all switch states and sizes.


Example: HTML inline code <div aria-checked="false" role="switch">  <input type="switch" id="accent_normal">  <label data-css-mark for="accent_normal">   unchecked  </label> </div> <div aria-checked="true" role="switch">  <input type="hidden" value="true">  <label data-css-mark>   checked  </label> </div>

Differnt Colors

You can combine both primary and secondary colors in the same switchbutton.

Example: HTML inline code <div data-css-role="switch">  <input name="different" id="different_1" type="switch">  <label data-css-accent data-css-mark for="different_1">   Primary and accent  </label> </div> <div data-css-accent data-css-role="switch">  <input name="different" id="different_1" type="switch">  <label data-css-primary data-css-mark for="different_1">   Accent and primary  </label> </div>

Notice: As mentioned everywhere, we take primary color as the default, so there's no need to declare that the component uses the primary color for the button in the combination "Primary and accent".
