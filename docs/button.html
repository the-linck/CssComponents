Plain Button

Every app or website will eventually need buttons, isn't true?
To supply this need there's the Button Component, that allows a very flexible use of buttons atop of HTML5, ARIA and some custom attributes.

A simple button may be created with any HTML element, but it's a good pratice to use only elements that interect natively with click/touch events, like:

<a>
Example: HTML inline code <button data-css-role="button" title="Button²">&lt;button&gt;</button> <a role="button" title="Link button">&lt;a&gt;</a> <input role="button" title="Input button" type="button" value="&lt;input&gt;"/>

Sizes

We offer five button sizes, wich can be chosen by our data-css-size attribute:

Example: HTML inline code <button data-css-size="small" role="button">small</button> <button role="button">standart (medium)</button> <button data-css-size="big" role="button">big</button> <button data-css-size="huge" role="button">huge</button <button data-css-size="full" role="button">fullsize</button

States

We provide four button states:

  • normal
  • current (selected)
  • pressed
  • disabled

They are automatically chosen from the button current state in the page, but can also be forced by attributes.
You can use the ARIA attributes to force a button state.

Using ARIA states:
Example: HTML inline code <button data-css-role="button" >no state</button> <button data-css-role="button" aria-selected="true">selected</button> <button data-css-role="button" aria-pressed="true">pressed</button> <button data-css-role="button" aria-disabled="true">disabled</button>

Notice: You can also use the native disabled attribute to disable a button, of course.

Actions

Sometimes you need to highlight actions on your page, for that porpuse we provide the following types:

Example: HTML inline code <button data-css-role="button" data-css-action="primary">primary</button> <button data-css-role="button" data-css-action="secondary">secondary</button> <button data-css-role="button" data-css-action="tertiary">tertiary</button> <button data-css-role="button">common</button> <button data-css-role="button" data-css-action="special">special</button>

Groupped Buttons

Buttons alone can be very useful, but groupped they may be even more.
There's a lot of room for groupped buttons in today's apps, but little creativity in this article's author to make examples, so let's do a simple button group:

Split out the bone
Example: HTML inline code <div role="group" data-css-group="button">  <button role="button">Sp<sub>l</sub>it</button>  <button role="button">out</button>  <button role="button">the</button>  <button role="button">bone</button> </div>

Double Button

It's very common to have a group of two buttons, one with text and another wit an icon.
That's easy to achieve:

Example: HTML inline code <div role="group" data-css-group="button">  <button role="button">We're twins</button>
  <button role="button">&dtrif;</button> </div>
