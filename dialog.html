Standart Dialog

Dialogs are great elements to show messages, confirmations or even just separate content from the page.
Our implementation is quite simple, yet provides what most people will need.

For a simple dialog that will not block user's interaction with the page, you can use the Dialog Component with the dialog role.

Open Standart Dialog

✖ I'm a standart dialog

And i am the dialog's text.
Example: HTML inline code <a href="#standart-dialog" id="standart-button" role="button">  Open Standart Dialog </a> <aside aria-labelledby="standart-header" id="standart-dialog" role="dialog">  <h5 data-css-role="heading" id="standart-header">   <a href="#standart" data-css-close data-css-float="right" title="Close">&#x2716;</a>   I'm a standart dialog  </h5>  <div>   And i am the dalog's text.  </div> </aside>

Notice several details about the implementation above:

  • The dialog is openend by a link button, with it's href pointing to the dialog's id.
    We show Dialogs by the :target selector, so the element declared as the dialog must be the current target of the page to show the dialog - wich is easy to do with links and ids.
  • The dialog is an aside element with role="dialog" and an ID
    A dialog is made to be apart of the page, so the aside element feels like a natural choice.
  • A header is present
    Accordingo to ARIA, dialogs must be labeled. The simplest way to it is to prive a header to the dialog, using it's title as a label.
  • The close button points to the button wich opens the dialog
    We recommend to always point the dialog's close button to the element wich opened the dialog or to the section of the page where it is. With that rule we avoid much unnecessary scroll on the page.

Modal

If you must make the user interact only with the contents of the dialog, you can choose the alertdialog role (as alert dialogs are always modal) or add our data-css-modal attribute to the dialog
